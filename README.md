# Hidden Markov Models — Human Activity Recognition

Formative 2. Inferring hidden human-activity states (**still, standing, walking, jumping**) from
smartphone **accelerometer** and **gyroscope** signals with a Hidden Markov Model (Baum–Welch
training, Viterbi decoding).

## Running

```bash
pip install -r requirements.txt
jupyter notebook HMM_Activity_Recognition.ipynb   # Run All
```

The notebook auto-adds a local `.pylibs/` folder to `sys.path` if present (this repo's offline
dependency cache); inside a normal virtualenv that line is a harmless no-op.

## Data facts

- **Device / app:** single smartphone, Physics Toolbox.
- **Sampling rate:** ~98 Hz on every file (measured); resampled to a uniform 100 Hz grid.
- **Sensors:** accelerometer `ax,ay,az` (m/s²); gyroscope `wx,wy,wz` (rad/s).
- Accelerometer and gyroscope were logged in separate sessions per activity, so features are
  paired by window index (see the notebook's feature-extraction section).

## Result

The from-scratch Viterbi decoder matches `hmmlearn`, Baum–Welch converges via a log-likelihood
tolerance check, and the model separates the four activities on the held-out windows. See the
notebook and [`figures/`](figures/) for the transition heatmap, decoded sequence, and confusion matrix.
